CRYPTOTAB BROWSER — EMBEDDED HOSTNAMES / URLS
=============================================

Extraction:
  strings -a -n 8 "<framework_binary>"      | grep -oE 'https?://[A-Za-z0-9.-]+(/[A-Za-z0-9._~:/?#@!$&+,;=%-]*)?' | sort -u
  strings -a -n 8 "<resources.pak>"         (then GRIT-decompressed)
  + the carved cryptotab/loader.js + cryptotab/tab.js JS bundles from resources.pak

Total: 663 unique URL prefixes in the framework binary. The vast majority are
upstream Chromium boilerplate (search-engine suggest URLs, OCSP CRL servers,
DoH endpoints, Google services). Below are the URLs that matter to this
investigation.

mining-protocol.md.

                                                      Google component update

HOWEVER: there is NO ChromiumUpdater.app bundled in this .app, no `ksadmin`,
no Sparkle, no Keystone. Even though the URLs are referenced in the binary
they cannot actually be used to update the browser — Chromium's updater
binary is missing. CryptoTab almost certainly auto-updates by some other
mechanism (their `/api/v2/platform/pc/app` JSON returns version data plus a
URL to a remote JS payload that's eval()'d; see mining-protocol.md). The
Google update URLs are leftover/dead strings from upstream Chromium code paths.

Crash reporting still goes to:
https://clients2.google.com/cr/report      crashpad / breakpad
https://content-autofill.googleapis.com    Autofill server
... and several other standard Chromium-internal Google endpoints.
